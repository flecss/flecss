flecss

flecss: Simple and minimalist CSS framework built on the power of flexbox.

Unlike Bootstrap or Tailwind, flecss is not as granular as CSS itself.
Instead, it follows a minimalist approach: Each class bases on a style requirement common for the majority of web-based designs1.
Anything else is left to individual stylesheets.

Integration

From CDN Recommended

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/flecss/dist/flecss.css">

From NPM

npm install -D flecss
ℹ️   Certain CSS resets are inherent to a flecss integration (see _reset.scss).

Classes

The flecss class anatomy consists of a purposeful classifier A, and possibly a specifier B: .A_B. An unspecific class – i.e. without a specifier suffix – represents the default style. Each class also associates with a handy shorthand classifier.

ℹ️   Unlike for BEM, specific classes are standlone, i.e. inherit all shared style foundations. It is thus not required to assign the joint default and specific class name ❌ .p.p_top ✅ .p_top.

.d .display

The display class represents toggleable display conditions. At that, the explicit class describes the less common condition, whereas absence implies the complementary (common) condition.

_disable

Greyout and disable for interaction.

_hide

Hide, but keep layout.

_collapse

Hide, including layout (bounding box).

Space Modifier

Instead of a specifier, a double dash indicated modifier C can be used to override the pivotaö space property: .A--C. The space modifier is therefore linked with the global space variable. For instance, using .m.m--large would result in subsequently larger margin compared to just .m (which equals .m.m--medium).

ℹ️   The space modifier simply overrides the pivotal space variable for a respective purpose. The modifier is in fact not standalone like specified classed. As for BEM, it is thus required to assign the joint class and modified class name ❌ .p--large ✅ .p.p--large.
--xs --extra-small

Extra small spacing.

--s --small

Small spacing.

--m --mediumdefault

Medium spacing.

--l --large

Large spacing.

--xl --extra-large

Extra large spacing.

The size of a space unit (e.g. --s) scales progressively aroud the --m (≡ --space) by a factor --space-factor. For instance, --xl corresponds to --space * --space-factor^2 – i.e. 4rem by default.

--space 1rem
--space-factor 2.0

Font sizing in flecss behaves analogous to the above described spacing. Heading sizes scale quadratically to --font-factor with respect to their level.

--font-size 16px ≡ 16px
--font-factor 1.5

Build Interface

Integrated via NPM, flecss comes with a mature build interface.

ℹ️   The flecss build interface bases on the sass transpiler and clean-css optimiser.
ℹ️   Working with flecss on the SCSS layer enables custom overrides of the global flecss variables (e.g. --space).

CLI Recommended

npx flecss [--]*
Flag Description
--standalone -S

Build without including flecss.

--watch -W

Watch file changes for incremental builds.

API

interface IBuildOptions { isDevelopment?: boolean; isStandalone?: boolean; } function buildCSS(sourcePath: string, targetPath?: string, options?: IBuildOptions & { modTimeTolerance?: number; }): Promise<{ executionTimeMs: number; sourcePath: string; targetPath: string; targetSizeByte: number; }> function transpile(sourcePath: string, targetPath: string, options?: IBuildOptions): { css: string; loadedUrls: string[]; }

Example

const flecss = require("flecss"); flecss.buildCSS("./app.scss"); // target path: app.css

Utilities

The flecss build interface predefines a set of helpful utility mixins (flecss_<*>) and functions (flecss-<*>) that can be used throughout custom SCSS code.
